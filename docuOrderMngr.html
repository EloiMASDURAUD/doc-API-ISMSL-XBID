DOCUMENTACION OrderMngr

OrdersParams :

OrdersParams es una clase que define parametros de ordenes. tiene 5 parametros

- orderPosicion

- delta

- executionCondition

- step

- ppd

orderPosicon es de tipo PosicionEnergia, define la posicion de las up en la que los ordenes deben resultar.

delta es el delta de precio applicado al precio maximo (o minimo) de compra (o venta). delta se anade a un orden de venta y se quita a un orden de compra. (delta positivo = orden que no casa ahora, delta negativo = orden que casa ahora).

executionCondition ver documentacion OMIE.

step debe ser null para ordenes simple, sino es la cantidad de un tramo de iceberg. step debe ser mayor de 5 y orderPosicion hecho de manera que posicionUp - orderPosicion > step sino no se envia ordenes (para este contracto).

ppd es la variacion de precio cada tramo de un iceberg. Para venta, ppd = -ppd.abs(); para compra, ppd = ppd.abs(). Entonces no hace falta el signo de ppd.

EJEMPLO :

para un contracto, si la up esta a -10MWH y orderPosicon a -22MWH, step a 5 y ppd a 2. Si llamamos minPrc el precio minimo de venta para este contracto, el orden resultara en :

- un orden iceberg de compra con un primero tramo de 5MWH a minPrc,

- un secundo tramo de 5MWH a minPrc a minPrc-2,

- un ultimo tramo de 2MWH a minPrc a minPrc-4.

PosicionEnergia

Esta clase representa una posicion para los contractos abiertos en MWH de nuestras up. se utiliza en OrdersParams para el calculo de ordenes.

EJEMPLO :

Un objeto PosicionEnergia se crea con este metodo :

xbidInfo.newPosicion(ISMSC01, ISMRE02, ISMRE03);

Donde ISMSC01, ISMRE02, ISMRE03 son double, la posicion de cada up para cada contracto (no sepuede hacer una posicion distincta cada contracto ahora)

Para poner todas las up a zero, hay que dar este PosicionEnergia a OrdersParams :

xbidInfo.newPosicion(0., 0., 0.); //los . son porque los parametros deben ser double

Para comprar 10MWH cada contracto solo para ISMSC01, hay que dar este PosicionEnergia a OrdersParams :

xbidInfo.getPosicion().subtract(xbidInfo.newPosicion(0.1, -0.1, -0.1));

OrderMngr.addOrders

Este metodo guarda ordenes en un set para despues enviar los al servidor como NewOrders con el metodo sendNewOrders

El unico parametro esta un objeto OrdersParams crea un orden simple o iceberg (depiende de los parametros) para cada contracto y up en la orderPosicion de los parametros si el orden esta valido para este contracto/up

Los ordenes estan anadido en el set newClientOrders de OrderMngr y se quitan dependiente a lo que llega en la cola

EJEMPLO :

Ordenes simple para poner todas las up a 0MWH con un orden para cada up/contracto al precio maxCompra/minCompra :

PosicionEnergia posicion = xbidInfo.newPosicion(0., 0., 0.);

OrdersParams params = new OrdersParams();

params.setOrderPosicion(posicion);

orderMngr.addOrders(params);

Ordenes para comprar 10MWH con ISMSC01 cada contracto a minVenta - 5

PosicionEnergia posicion = xbidInfo.getPosicion().subtract(xbidInfo.newPosicion(10., 0., 0.));

OrdersParams params = new OrdersParams();

params.setOrderPosicion(posicion);

params.setDelta(5.);

orderMngr.addOrders(params);

Ordenes para comprar 12MWH con ISMSC01 cada contracto a minVenta con un iceberg de tramos de 5MWH y un ppd a 2

PosicionEnergia posicion = xbidInfo.getPosicion().subtract(xbidInfo.newPosicion(12., 0., 0.));

OrdersParams params = new OrdersParams();

params.setOrderPosicion(posicion);

params.setPpd(2);

params.setStep(5)

orderMngr.addOrders(params);

OrderMngr.sendNewOrders

Este metodo envia los nuevos ordenes en el set newClientOrders con el management NewOrders.

El metodo sendNewOrders(String complexCondition, int basketSize, boolean sendAll) tiene 3 parametros :

- complexCondition es la condicion compleja de losbaskets, ver la documentacion de OMIE.

- basketSize esta el tamano de los baskets enviados, si esta mayor de 100 los baskets todavia seran de 100 ordenes.

- sendAll, si true se envia todos los ordenes en el set newClientOrders sino se envia solo el primero basket.

Si los ordenes que se han enviado vuelven como valido en la cola, se quitan del set newClientOrders sino se quedan en el set para reintentar enviar los.

OrderMngr.cancelClientOrders

Este metodo pone todos los ordenes activo del cliente en el set cancelClientOrders para despues cancelarlos con el metodo sendCancelOrders.

No necesita parametros y solo se puede poner todos los ordenes en el set por ahora.

Si en el codigo se envian ordenes antes de usar cancelClientsOrders para cancelar estos ordenes que acaban de ser enviado, esta recomandado poner un tiempo de espera asi que los ordenes nuevos llegan a la cola y que se actualiza los datos del API. Sino esta probable que solamente se anaderon algunos o ningunos ordenes al set cancelClientOrders.

EJEMPLO :

mala practica

orderMngr.addOrders(params);

orderMngr.sendNewOrders("None", 100, true);

orderMngr.cancelClientOrders();

buena practica

orderMngr.addOrders(params);

orderMngr.sendNewOrders("None", 100, true);

Thread.sleep(3000);

orderMngr.cancelClientOrders();

OrderMngr.sendCancelOrders

Este metodo envia los ordenes en cancelClientOrders para cancelar con el Management CancelOrers.

como sendNewOrders se puede elegir el tamano de basket enviado y si se envia solo un basket

Si un orden llega en la cola como no mas valido, se quita del set cancelClientOrders.

OrderMngr.updateAllClientOrders

Este metodo modifica todos los ordenes del cliente con los parametros.

Similario a addOrders tiene como unico parametro un objeto OrdersParams.

Itera sobre cada orden en el orderlist y guarda ordenes modificado en el set modifyClientOrders. Solo modifica la posicion y el precio del orden acordado con el parametro.

No se puede cambiar el typo de orden, solo la cantidad y el precio (ppd y tQty tambien para icebergs).

OrderMngr.sendModifyOrders

No tiene parametro, envia los ordenes modificado en el set modifyClientOrders al servidor con el management ModifyOrders uno a uno.

Si la modificacion esta valida o que el orden no existe mas (casado o cancelado), se quita el orden del set.

EJEMPLOS DE CODIGO PARA LA PRUEBA

Esos son codigos que se pueden copiar para las differentes pruebas.

Oferta de compra simple que no case :

Envia para cada contracto/up un orden de compra de 0.1 MWH con un delta de 300, asi no case el orden. Los ordenes se envian uno a uno.

PosicionEnergia posicion = xbidInfo.getPosicion().subtract(xbidInfo.newPosicion(0.1, 0.1, 0.1));
OrdersParams params = new OrdersParams();
params.setOrderPosicion(posicion);
params.setDelta(300.);
orderMngr.addOrders(params);
orderMngr.sendNewOrders("None", 1, true);

Poner false en sendNewOrders si quieres enviar solo un orden en total.

Modificación de una oferta simple :

Envia para cada contracto un orden simple para poner ISMSC01 a 10MWH con un delta de 200 (asi no case). Despues los modifica para poner ISMSC01 a 20MWH con un delta de 0.

PosicionEnergia posicion = xbidInfo.newPosicion(-10., 0., 0.);
OrdersParams params = new OrdersParams();
params.setOrderPosicion(posicion);
params.setDelta(200.);
orderMngr.addOrders(params);
orderMngr.sendNewOrders("None", 100, true);
Thread.sleep(3000);
posicion = xbidInfo.newPosicion(-20., 0., 0.);
params = new OrdersParams();
params.setOrderPosicion(posicion);
params.setDelta(0.);
orderMngr.updateAllClientOrders(params);
orderMngr.sendModifyOrders();
